# Problem: The Useless Support Inbox

> "We get a lot of inbound questions about our products. Right now someone reads them manually and replies. It takes forever and half the answers are already in docs we have. Fix this. We don't care how."  
How will you fix this? Tell us the solution "Don't Build".

## How I would solve this issue

Before building any kind of AI chatbot, I’d first add **structure**, because structure alone can solve a big part of the problem.

**1) Add a proper ticket + category system**  
I would create clear categories like **Billing**, **Setup**, **Bugs**, **Feature Requests**, etc., and make sure every incoming query gets tagged into one of these. This immediately makes routing faster and reduces the “someone has to read everything” problem.

**2) Build a simple, organized FAQ / Help Center**  
Once the common categories are defined, I’d create an FAQ section with the same structure. Most support questions repeat — if the answers are already in docs, they just need to be easy to find.

**3) Only after that, add AI (as a last layer, not the first)**  
If users still can’t find what they need, then I’d add an AI chatbot — but not as a replacement for support. I’d use it mainly for:
- guiding users to the right FAQ/doc page  
- and if it still can’t solve the issue, **collecting the right details and creating a ticket** automatically (so the support team gets a clean, structured request)

Then the support team can follow up through email with everything already captured properly.

This way, the inbox gets faster and cleaner **without immediately jumping to building a full AI support system**.
